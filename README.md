## Project Structure

### Lab 1: Design Patterns
Independent Python implementations of software design patterns.
* **Abstract Factory:** Creates families of related objects without specifying their concrete classes.
* **Composite:** Composes objects into tree structures to represent part-whole hierarchies.
* **Iterator:** Provides a method to access the elements of a complex aggregate object sequentially.

### Labs 2–4: Fullstack Development & Testing
Development of the core SmartHome application architecture.
* **Lab 2 (Backend):** RESTful API built with **FastAPI**. Uses the **uv** package manager for dependency resolution.
* **Lab 3 (Frontend):** Client-side application built with **React.js** to interact with the backend API.
* **Lab 4 (Testing):** Asynchronous testing suite built with **Pytest**. Includes code coverage tracking.

### Lab 5: DevOps & Cloud Deployment
System integration and cloud infrastructure setup.
* **Containerization:** Backend environment isolated using a **Dockerfile**.
* **Cloud Hosting:** API deployed as a container unit on **Render**; client application deployed on **Vercel**.
* **CI/CD:** Automated pipeline configured via **GitHub Actions** to trigger tests and deployments on every push to the main branch.
